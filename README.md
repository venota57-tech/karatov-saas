# KARATOV CX Hub — v3-27 WB Live Route Viewer

Главное изменение: модуль FBO теперь работает по стратегии **ЛК WB-first**.

- POSTAVLENO Web отключен полностью.
- WB API не используется как основной мониторинг окон, поэтому 429 не должен останавливать автопилот.
- WB API оставлен только как диагностический блок.
- Основной контур: Playwright открывает ЛК WB, проверяет целевые даты, делает скриншот и готовит бронь в safe-mode.
- Боевые клики бронирования заблокированы до подтверждения карты экранов WB.

Запуск:

```bash
docker compose down
docker compose up --build
```

Открыть: http://localhost:5173

Порядок подключения ЛК WB:

1. Автобронирование FBO → ЛК WB / Playwright safe-mode.
2. Нажать «Открыть ЛК WB».
3. Авторизоваться в кабинете WB на скриншоте/странице Playwright.
4. Нажать «Проверить вход».
5. Нажать «Мониторить окна через ЛК WB».
6. После снятия реальных экранов и подтверждения сценария можно будет включить боевой режим бронирования.

## v3-27: WB Live Route Viewer

Добавлено для FBO:
- Live View ЛК WB: скриншот Playwright обновляется каждые 2 секунды в интерфейсе.
- Статус текущего шага и прогресс маршрута создания поставки.
- Кнопка остановки маршрута.
- Настройка ожидаемого ЛК WB и распознавание юрлица на экране.
- Текстовое подтверждение перед финальной кнопкой “Создать поставку”.
- Боевой режим не нажмет финальную кнопку, если подтверждение не совпадает с формулой `Создавать поставки в <ожидаемый ЛК>`.
